# Water Ops MVP

Русская MVP-система для операционного учета бизнеса по продаже и доставке бутилированной воды.

Система состоит из:

- админ-панели владельца на Next.js;
- Telegram-бота для сотрудников;
- схемы Supabase/Postgres для ежедневных отчетов;
- логики сводки, кассы и простых расхождений.

## Что уже есть

- Дашборд: приход, продажи/отгрузки, расходы, касса, остатки, павильоны, кулеры, расхождения.
- Брак и списание: отдельный учет бутылей/бутылок, которые нельзя продать или вернуть в оборот.
- Мобильная версия админки: таблицы превращаются в читаемые карточные списки, навигация становится горизонтальной.
- Разделы: сотрудники, склады, товары, павильоны, доставки, приход, остатки, расходы, кулеры, отчеты, касса.
- Telegram-бот на русском: кнопки и пошаговые сценарии для сотрудников.
- Демо-режим без Supabase: админка открывается с тестовыми данными.
- Supabase-режим: после заполнения `.env.local` данные читаются из базы.

## Стек

- Next.js
- TypeScript
- Tailwind CSS
- Supabase
- Telegram Bot API через Telegraf

## Быстрый старт

```bash
npm install
cp .env.example .env.local
npm run dev
```

Админка будет доступна по адресу:

```text
http://localhost:3000
```

Если Supabase-переменные не заполнены, приложение работает в демо-режиме.

## Деплой, чтобы работало без включенного ПК

Самый быстрый бесплатный вариант для текущей мобильной версии - Render или Railway как Node web service.

### Render

1. Создайте аккаунт на Render.
2. Загрузите проект в GitHub.
3. В Render выберите `New` -> `Blueprint`.
4. Подключите репозиторий.
5. Render прочитает `render.yaml` и запустит сервис командой:

```bash
npm run mobile
```

После деплоя ссылка будет вида:

```text
https://voder-mobile.onrender.com/employee
```

### Railway

1. Создайте аккаунт на Railway.
2. Создайте новый проект из GitHub.
3. Railway использует `Procfile`:

```text
web: npm run mobile
```

4. В настройках сервиса включите публичный домен.

### Важно про данные

Без Supabase данные хранятся в памяти сервера. Для теста интерфейса это нормально, но для реальной работы нужно подключить Supabase, иначе при перезапуске бесплатного сервера отчеты могут сброситься.

Для постоянного хранения заполните переменные окружения на хостинге:

```bash
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
```

## Настройка Supabase

1. Создайте проект в Supabase.
2. Откройте SQL Editor.
3. Выполните файл:

```text
supabase/schema.sql
```

4. Заполните `.env.local`:

```bash
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
```

`SUPABASE_SERVICE_ROLE_KEY` нужен серверной части и Telegram-боту для записи отчетов. Не публикуйте его в браузере.

## Telegram-бот

Создайте бота через BotFather и добавьте токен:

```bash
TELEGRAM_BOT_TOKEN=123456:telegram-bot-token
```

Локальный polling-режим:

```bash
npm run bot
```

Webhook-режим для продакшена:

```bash
TELEGRAM_WEBHOOK_URL=https://your-domain.com/api/bot/webhook
TELEGRAM_WEBHOOK_SECRET=change-me
npm run telegram:webhook
```

### Кнопки бота

- Приход товара
- Продажа на склад
- Продажа на павильон
- Остаток
- Расход
- Брак / списание
- Отчет по павильону
- Кулер
- Завершить смену

### Сценарии бота

Приход товара:

- склад / локация;
- тип товара;
- количество;
- тип закупки: безнал или наличка;
- сумма считается автоматически: безнал по 5 руб., наличка по 115 руб.

Продажа на склад:

- склад / клиент;
- тип товара;
- количество проданных бутылок;
- цена считается автоматически: 250 руб. за бутылку;
- комментарий.

Продажа на павильон:

- павильон;
- количество проданных / оставленных бутылок;
- сколько бутылок забрал обратно;
- цена считается автоматически: 300 руб. за бутылку;
- комментарий.

Остаток:

- склад / машина / локация;
- тип товара;
- фактический остаток.

Брак / списание:

- склад / машина / павильон;
- тип товара;
- количество брака;
- причина;
- комментарий.

Расход:

- категория;
- сумма;
- тип оплаты;
- комментарий.

Павильон:

- павильон;
- продано / передано;
- заметки.

Кулер:

- павильон;
- наш кулер / не наш кулер;
- комментарий.

## Бизнес-логика MVP

Дашборд считает:

- сколько товара пришло;
- сколько продано;
- сколько бутылок забрали обратно с павильонов;
- сколько потрачено;
- сколько бутылок списано как брак;
- сколько денег должно остаться;
- сколько наших кулеров стоит на павильонах;
- какие расхождения и проверки требуют внимания.

## Важные файлы

- `src/app/page.tsx` - главный дашборд.
- `src/app/dashboard/[section]/page.tsx` - разделы админки.
- `src/components/app-shell.tsx` - desktop/mobile навигация.
- `src/components/ui.tsx` - таблицы, карточки, алерты.
- `src/bot/bot.ts` - Telegram-бот.
- `src/bot/flows.ts` - русские сценарии бота.
- `src/bot/store.ts` - запись в Supabase и итог смены.
- `supabase/schema.sql` - схема базы.
- `supabase/defective-write-offs-migration.sql` - миграция для добавления брака в уже созданную базу.
- `supabase/fixed-prices-migration.sql` - миграция для фиксированных цен продаж и закупок.
